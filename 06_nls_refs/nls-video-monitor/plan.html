Architectural approach

1. Two-layer scheduling (not browser cron)

LayerRoleWhen to use
In-process runner Background thread inside gravity-serve (or standalone nls-video cron-runner) polls due tasks every 30–60s Dev, always-on dashboard
System crontab */5 * * * * nls-video cron tick as a safety net / headless mode Production, reboot survival

2. Task registry

cron_tasks (
  id, name, task_type,   -- mint_watch | pos_reconcile | avd_scan
  schedule_cron,
  enabled, last_run, next_run, last_status, last_error, config_json
)
cron_runs (id, task_id, started, finished, status, result_json)
  • mint_watch — every 10 min: scan ~/Downloads/direct-h264-*, mint if missing
  • pos_reconcile — every 30 min: merge vertex + storage_memory_category into position index
  • avd_scan — every 15 min: tag analog files with AVD records (probe hash, asc_mode=FILMIC)

3. Position index (canonical POS artifact)

{
  "id": "sha256-or-pattern",
  "avd": { "path": "~/Downloads/direct-h264-….mp4", "asc_mode": "FILMIC", "probe": {} },
  "pos": { "vertex": [0,1,0,1], "pattern_code": "AB.2:4.P&B.F1", "storage_memory_category": { "zone": "default-zone" } },
  "synced_at": "ISO8601"
}

Conflict rule: blockcode DB wins for vertex; visualist catalog wins for storage_memory_category; AVD path from newest file mtime.

4. Task implementations

TaskReusesDoes
mint_watchinterlaterus_desktop.mint_media_fileIdempotent mint for new downloads
avd_scanprobe_media, _recent_downloads_listBuild/update AVD side of position_index
pos_reconcileget_storage_memory_category, occupied_verticesAlign POS fields; flag orphans

5. API surface

MethodPathPurpose
GET/api/cronList tasks + last run status
GET/api/cron/runsRecent run log (tail 50)
POST/api/cron/runManual trigger { "task_id": N }
POST/api/cron/toggleEnable/disable task
GET/api/positioningFull position_index (AVD/POS pairs)
GET/api/positioning/orphansItems missing AVD or POS half

6. HTML stack changes

  • Scheduled tab — task table, Run now, Enable/Disable
  • Positioning tab — AVD (ASC filmic) | POS (vertex + zone), orphan badges, optional 4×4 vertex heatmap
  • ASC Live — vertex + full storage_memory_category per catalog row
